1. Overview

VitalGuard AI is a single-file, infrastructure-independent, offline-first safety framework. Its core loop uses Web Bluetooth BLE scanning to collect tag RSSI values, then applies filtering and interpretation layers such as Kalman smoothing, distance estimation, and lightweight on-device learning to determine zone states and trigger alerts, guidance, and SOS workflows.

2. Execution, Permissions, and Offline Rules

  • Recommended: Run under HTTPS (local server or static hosting) for the most reliable Web Bluetooth and PWA behavior.
  • file://: Depending on browser policy, Web Bluetooth and Service Worker features may be partially restricted.
  • Offline rule: v4.2.3 uses CSP (connect-src 'none') to block unintended network calls.

8. v4.2.3 Security Hardening Summary

The four most important SecurityLab comments are reflected in v4.2.3 as follows.

  1. Service Worker cache→network fallback removed: switched to a cache-only strategy.
  2. Session-end auto-wipe option: autoWipeOnClose toggle + Security.fullWipe().
  3. CSP meta added: includes connect-src 'none' to block network egress.
  4. prompt() removed: encrypted-backup passphrase entry now uses the SecurePrompt modal.
v4.2.3 also adds a 20–50 ms jitter to BLE restart delay to reduce deterministic timing patterns. Because the app listens for BLE traffic rather than transmitting custom payloads, payload-padding suggestions are only partially applicable.
